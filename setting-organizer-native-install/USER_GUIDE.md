# Setting Organizer 使用说明

版本：`0.2.0`

## 安装

将整个 `setting-organizer` 文件夹放到 SillyTavern 第三方扩展目录：

```text
SillyTavern/public/scripts/extensions/third-party/setting-organizer
```

目录中应包含：

```text
manifest.json
index.js
style.css
src/
```

重启 SillyTavern 或刷新浏览器页面后，在扩展设置区域找到“设定整理器”面板。

## 基础使用流程

1. 打开 SillyTavern。
2. 打开扩展设置中的“设定整理器”面板。
3. 在输入框粘贴设定文本，或点击“读取当前聊天”。
4. 选择整理目标：
   - 角色卡
   - 世界书
   - 两者都选
5. 选择分析模式：
   - 首次测试建议使用 `模拟结果`。
   - 真实整理时使用 `当前 SillyTavern 模型`。
6. 选择 token 预算模式。
7. 点击“开始分析”。
8. 在结果页检查总览、角色、世界书和警告。
9. 编辑草稿，删除不需要的条目。
10. 根据需要导出、备份或创建到 SillyTavern。

## 读取当前聊天

点击“读取当前聊天”后，插件会读取当前 SillyTavern 会话并填入输入框。

可选范围：

- 最近 20 条
- 最近 50 条
- 全部
- 手动索引，例如 `0, 2, 5-8`

注意事项：

- 插件不会自动读取聊天，必须手动点击。
- 读取结果会覆盖当前输入框内容。
- 当前聊天为空或接口不可用时，会显示 `E012`。
- 诊断日志只记录范围、条数和长度摘要，不记录完整聊天正文。

## 检查分析结果

结果页包含四个页签：

- `总览`：角色草稿数、世界书条目数、警告数、粗估 token。
- `角色`：编辑角色草稿字段。
- `世界书`：编辑世界书条目、关键词和启用状态。
- `警告`：查看重复内容、重复关键词和 token 预算提醒。

建议在创建到 SillyTavern 前先检查：

- 角色名是否正确。
- 世界书关键词是否过宽或过窄。
- 世界书条目是否启用。
- 是否存在重复标题、重复正文或 token 超预算警告。

## 导出草稿

结果页支持导出：

- 完整草稿。
- 角色草稿。
- 世界书草稿。
- SillyTavern 角色格式。
- SillyTavern 世界书格式。

导出文件由浏览器下载机制保存。Android / MuMu 中的保存位置由浏览器决定。

## 创建备份

点击“创建备份”会把当前草稿保存到浏览器 localStorage。

备份用途：

- 记录写入前草稿。
- 辅助后续手动恢复或排查。

限制：

- 备份不会自动上传。
- 备份不是完整自动回滚。
- 清理浏览器数据可能会删除 localStorage 备份。

## 创建到酒馆世界书

1. 确认结果页存在启用的世界书条目。
2. 点击“创建到酒馆世界书”。
3. 查看步骤报告：
   - 校验世界书草稿
   - 创建导入前备份
   - 创建新世界书
   - 验证旧世界书未变化
4. 成功后，使用 SillyTavern 原生世界书管理器继续编辑。

如果世界书已重名，插件会停止创建，避免覆盖已有数据。

## 创建到酒馆角色

1. 确认结果页存在角色草稿。
2. 点击“创建到酒馆角色”。
3. 查看步骤报告：
   - 校验角色草稿
   - 创建导入前备份
   - 创建新角色
   - 验证旧角色未变化
4. 成功后，在 SillyTavern 原生角色面板继续管理角色。

## 创建角色并绑定本次世界书

如果需要让新角色关联本次新建世界书：

1. 确认结果页同时存在角色草稿和启用的世界书条目。
2. 勾选“创建角色后绑定本次新建世界书”。
3. 点击“创建到酒馆角色”。
4. 查看步骤报告。

该流程会多执行两个步骤：

- 创建待绑定世界书。
- 绑定角色到新世界书。

如果绑定失败，会显示 `E013`。此时角色或世界书可能已经创建成功，需要根据步骤报告和诊断日志确认，再在 SillyTavern 原生角色面板手动绑定。

## 日志与排障

插件有日志功能。面板提供：

- `导出诊断日志`
- `清空诊断日志`

建议排障流程：

1. 复现问题。
2. 点击“导出诊断日志”。
3. 记录错误码和操作步骤。
4. 如果是创建或绑定失败，查看步骤报告，确认失败点。
5. 如果是兼容性错误，查看 `setting-organizer/API_COMPATIBILITY.md`。

日志会记录：

- 扩展加载。
- 模型调用。
- 聊天读取。
- 导出。
- 备份。
- 世界书创建。
- 角色创建。
- 角色世界书绑定。
- 原生世界书入口。

日志会脱敏 API key、Cookie、token、secret、password 等字段。长文本只保存摘要，不保存完整聊天正文或完整 prompt。

## MuMu / Android 测试

MuMu 浏览器访问宿主机 SillyTavern 通常使用：

```text
http://10.0.2.2:8000/
```

建议测试顺序：

1. 确认本机 SillyTavern 服务已启动。
2. 在 MuMu 浏览器打开 `http://10.0.2.2:8000/`。
3. 确认扩展面板出现。
4. 使用 `模拟结果` 跑一轮分析。
5. 验证结果页、导出、本地备份、诊断日志导出。
6. 如需真实写入，使用 `SO_*` 前缀创建测试角色和世界书，并记录名称。

## 常见错误处理

页面错误提示会优先显示实际错误内容和处理建议，同时保留错误码。排障时建议同时记录“错误标题”和“错误码”。

| 错误码 | 处理方式 |
| --- | --- |
| `E001` | 检查 SillyTavern 当前模型连接。 |
| `E002` | 模型输出不是 JSON，重新分析或改用模拟结果测试。 |
| `E003` | 模型输出结构不符合草稿要求。 |
| `E004` | 输入内容不足或没有提取到目标内容。 |
| `E005` | 缩短输入或调高 token 预算。 |
| `E006` | 查看诊断日志，确认草稿能否序列化。 |
| `E007` | 检查 localStorage 是否可用或已满。 |
| `E008` | 检查角色创建 API 兼容性。 |
| `E009` | 检查世界书创建 API 和是否重名。 |
| `E010` | 当前 SillyTavern API 不兼容，需要重新做 runtime probe。 |
| `E011` | 停止后续写入，手动检查旧数据是否完整。 |
| `E012` | 当前聊天为空或不可读取，改用手动粘贴。 |
| `E013` | 角色世界书绑定失败，检查步骤报告并手动绑定。 |
