# Setting Organizer

SillyTavern 设定整理器扩展。

当前已完成到 `TC-12` 创建新角色。扩展提供基础面板、文本输入、整理目标选择、Token 预算模式、本地设置保存、分析状态、mock 角色草稿、mock 世界书草稿、基础编辑、JSON 解析、字段规范化、错误码、版本化整理 prompt、独立 token 粗估模块、基础警告规则、JSON 导出、本地备份记录、世界书导入报告和角色导入报告。世界书创建已在 SillyTavern 1.18.0 测试环境中通过 `saveWorldInfo(name, data, true)` 验证；角色创建已通过 `/api/characters/create` 验证。

## 安装

将 `setting-organizer` 目录放入 SillyTavern 的第三方扩展目录中，然后在 SillyTavern 中启用扩展。

## 当前功能

- 加载扩展入口。
- 显示基础设置面板。
- 输入设定文本。
- 选择整理目标：角色卡、世界书。
- 选择 Token 预算模式：轻量、标准、长篇、自定义。
- 使用浏览器本地存储保存输入和设置。
- 点击“开始分析”后生成 mock 角色草稿和世界书草稿。
- 展示总览、角色、世界书、警告标签页。
- 编辑角色字段、世界书标题、关键词和正文。
- 删除角色草稿或世界书条目。
- 启用 / 禁用世界书条目。
- 解析 JSON 或 Markdown 代码块包裹的 JSON。
- 对缺失字段、字符串关键词、异常 confidence 做规范化。
- 对非 JSON、顶层非对象、空结果返回错误码。
- 提供 `extract-setting-v0.1.0` Prompt 模板。
- 粗估输入、角色草稿、世界书草稿和总量 Token。
- 提示空名称、空标题、空正文、空关键词、短关键词、重复关键词、常驻过多和单条内容过长。
- 导出内部完整草稿、角色草稿、世界书草稿、SillyTavern 兼容角色 JSON、SillyTavern 兼容世界书 JSON。
- 创建本地备份记录，保存草稿、操作类型、目标信息、写入前状态占位和备份标识。
- 创建新世界书前自动创建备份。
- 创建新世界书后验证旧世界书名称仍保留。
- 创建新角色前自动刷新并记录已有角色摘要。
- 创建新角色后验证旧角色 avatar 仍保留。

## 已知限制

- 尚未接入 AI 模型调用。
- 当前 schema 文件已建立，但尚未引入完整 JSON Schema 引擎。
- SillyTavern 模型调用 adapter 已建立，但真实接口仍需在目标 SillyTavern 环境验证。
- 第一版备份只作为手动恢复依据，不承诺完整自动回滚。
- 世界书写入已在本地 SillyTavern 1.18.0 测试环境验证；其他版本仍建议先查看 `API_COMPATIBILITY.md`。
- 角色写入已在本地 SillyTavern 1.18.0 测试环境验证；其他版本仍建议先查看 `API_COMPATIBILITY.md`。
- 角色创建暂未实现新建世界书绑定。

## 安全策略

第一版默认不覆盖、不删除、不修改已有角色或世界书。AI 输出会先进入草稿和预览流程，写入前先创建本地备份记录，并在写入后对旧角色或旧世界书摘要做保留校验。
