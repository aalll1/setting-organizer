# Setting Organizer

SillyTavern 设定整理器扩展，用于把粘贴文本或当前聊天整理成可编辑的角色草稿和世界书草稿，再由用户确认后导出或安全新建到 SillyTavern。

当前完成到 `TC-15 README 与使用说明`。世界书创建已在 SillyTavern 1.18.0 测试环境中通过 `saveWorldInfo(name, data, true)` 验证；角色创建已通过 `/api/characters/create` 验证。其他 SillyTavern 版本请先查看 `API_COMPATIBILITY.md`。

## 安装

1. 将整个 `setting-organizer` 目录复制到 SillyTavern 第三方扩展目录：

```text
SillyTavern/public/scripts/extensions/third-party/setting-organizer
```

2. 确认目录中包含：

```text
manifest.json
index.js
style.css
src/
```

3. 重启 SillyTavern 或刷新浏览器页面。
4. 在 SillyTavern 扩展设置区域找到“设定整理器”面板。

## 基础使用流程

1. 在“粘贴设定文本”中输入设定、世界观、角色描述或剧情记录。
2. 或点击“读取当前聊天”，主动把当前聊天内容填入输入框。
3. 选择整理目标：角色卡、世界书，或两者都选。
4. 选择分析模式：
   - `模拟结果`：用于离线验证 UI 和导出流程。
   - `当前 SillyTavern 模型`：调用当前 SillyTavern 候选模型接口，目标环境必须兼容。
5. 点击“开始分析”。
6. 在结果页检查总览、角色、世界书和警告。
7. 编辑草稿字段，删除不需要的角色或世界书条目。
8. 根据需要执行：
   - 导出完整草稿 JSON。
   - 导出角色草稿 JSON。
   - 导出世界书草稿 JSON。
   - 导出 SillyTavern 兼容角色 JSON。
   - 导出 SillyTavern 兼容世界书 JSON。
   - 创建本地备份。
   - 预检导入世界书。
   - 预检导入角色。

## 当前聊天读取

- 必须用户点击“读取当前聊天”，扩展不会自动读取聊天。
- 支持读取范围：
  - 最近 20 条
  - 最近 50 条
  - 全部
  - 手动索引，例如 `0, 2, 5-8`
- 读取结果会覆盖输入框内容。
- 读取失败显示 `E012`，不会影响手动粘贴文本模式。
- 诊断日志只记录读取范围、条数和长度摘要，不记录完整聊天正文。

## 安全策略

- 默认不覆盖、不删除、不修改已有角色。
- 默认不覆盖、不删除、不修改已有世界书。
- 创建新角色或新世界书前会先创建本地备份记录。
- 写入后会校验旧角色或旧世界书摘要仍保留。
- 如果旧数据校验失败，会显示 `E011`，不能把写入视为完整成功。
- 备份只作为手动恢复依据，第一版不承诺完整自动回滚。
- AI 输出必须先进入草稿和预览流程，不能直接绕过用户确认写入 SillyTavern。

## 诊断日志

面板提供：

- `导出诊断日志`
- `清空诊断日志`

诊断日志会记录扩展加载、设置读写、分析、模型调用、导出、备份、角色创建、世界书创建和导入报告等事件。

隐私处理：

- 不记录 API Key、Cookie、认证 header、token、secret 或 password。
- prompt、聊天正文、角色正文、世界书正文等长文本只保留长度和短预览。
- 日志保存在浏览器 localStorage，最多保留 200 条。
- 日志不会自动上传，只有用户点击导出时才生成 JSON 文件。

## 错误码

| 错误码 | 含义 | 常见处理 |
| --- | --- | --- |
| `E001` | 模型调用失败 | 检查 SillyTavern 当前模型连接和 API 兼容性。 |
| `E002` | 模型输出不是合法 JSON | 重新分析，或改用模拟结果验证流程。 |
| `E003` | JSON Schema 校验失败 | 检查模型输出结构是否符合草稿格式。 |
| `E004` | 结果为空 | 输入内容可能不足，或模型没有提取到角色 / 世界书。 |
| `E005` | Token 超过预算 | 缩短输入，或切换更高预算模式。 |
| `E006` | 导出失败 | 查看诊断日志，确认草稿数据是否可序列化。 |
| `E007` | 创建备份失败 | 检查浏览器 localStorage 是否可用或已满。 |
| `E008` | 创建角色失败 | 查看诊断日志，确认 `/api/characters/create` 是否可用。 |
| `E009` | 创建世界书失败 | 查看诊断日志，确认 `saveWorldInfo` 是否可用。 |
| `E010` | 当前 SillyTavern 接口不兼容 | 查看 `API_COMPATIBILITY.md`，确认目标版本接口。 |
| `E011` | 旧数据完整性校验失败 | 停止后续写入，手动检查 SillyTavern 数据。 |
| `E012` | 当前聊天读取失败 | 确认当前页面存在聊天，或改用手动粘贴文本。 |

## 当前功能

- 扩展加载和基础面板。
- 粘贴文本输入。
- 当前聊天主动读取。
- 角色卡 / 世界书目标选择。
- Token 预算模式：轻量、标准、长篇、自定义。
- 本地设置保存。
- mock 分析结果。
- 候选 SillyTavern 模型调用 adapter。
- JSON / Markdown JSON 解析。
- 字段规范化和错误码。
- 角色草稿和世界书草稿编辑。
- 角色 / 世界书条目删除。
- 世界书条目启用 / 禁用。
- Token 粗估。
- 基础警告和规则级重复检测。
- JSON 导出。
- 本地备份记录。
- 安全新建世界书。
- 安全新建角色。
- 当前聊天读取。
- 运行诊断日志导出。

## 已知限制

- 真实模型输出质量取决于当前 SillyTavern 模型和 prompt 遵循情况。
- 当前 schema 文件已建立，但尚未引入完整 JSON Schema 引擎。
- SillyTavern 模型调用接口仍可能随版本变化。
- 世界书写入和角色写入只在本地 SillyTavern 1.18.0 测试环境验证。
- 角色创建暂未实现新建世界书绑定。
- 当前聊天读取已完成本地 adapter 和 UI 接入，但尚未在 MuMu 真实页面复测不同聊天范围。
- 诊断日志导出已完成本地实现，但尚未在 MuMu 真实页面复测文件下载。
- 重复检测只做规则级精确匹配，不做复杂语义推理。
- 不支持自动持续更新、RAG、关系图、高级冲突推理或完整自动回滚。

## Android / MuMu / Termux 注意事项

- MuMu 浏览器访问宿主机 SillyTavern 时通常使用：

```text
http://10.0.2.2:8000/
```

- 如果宿主机防火墙或端口占用导致无法访问，请先确认 SillyTavern 服务是否正在监听 `8000`。
- 移动端页面空间有限，建议先用“模拟结果”验证流程，再进行真实导入。
- Android 浏览器下载诊断 JSON 或导出 JSON 时，文件保存位置由浏览器决定。
- Termux 或 Android 环境中的路径和权限可能与 Windows 不同，扩展目录必须放在当前 SillyTavern 实际使用的 `third-party` 目录下。

## 维护建议

- 报错时先点击“导出诊断日志”，再记录操作步骤。
- 导入失败时保留备份标识和错误码。
- 如果 SillyTavern 升级后角色或世界书写入失败，先更新 `API_COMPATIBILITY.md`，再调整 `src/adapters/sillytavernApi.js`。
- UI、core、adapters、storage、prompts、schemas 已分层；后续修改应继续保持内部草稿格式和 SillyTavern 写入格式分离。
