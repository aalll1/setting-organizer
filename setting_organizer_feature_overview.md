# Setting Organizer 插件功能说明

版本：`0.2.0`  
适用环境：已在 SillyTavern 1.18.0 + MuMu Android 12 实测

## 插件定位

Setting Organizer 是 SillyTavern 的设定整理前置工作台。它把粘贴文本或当前聊天内容整理成可检查、可编辑、可导出的角色草稿和世界书草稿，再由用户确认后创建到 SillyTavern。

插件只做整理、校验、预览、导出、备份和最小安全创建，不替代 SillyTavern 原生角色管理器或世界书管理器。

## 核心功能

### 输入来源

- 手动粘贴角色设定、世界观、剧情记录或聊天整理素材。
- 主动读取当前 SillyTavern 聊天。
- 聊天读取范围：
  - 最近 20 条
  - 最近 50 条
  - 全部
  - 手动索引，例如 `0, 2, 5-8`

聊天读取必须由用户点击触发，插件不会自动读取聊天。

### 分析能力

- `模拟结果`：不调用模型，用于验证 UI、导出、备份和创建流程。
- `当前 SillyTavern 模型`：调用当前 SillyTavern 可用的生成接口。
- 支持角色卡、世界书或两者同时整理。
- 支持轻量、标准、长篇、自定义 token 预算。

### 草稿编辑

分析结果会进入草稿页，用户可以在写入前修改：

- 角色名称、描述、性格、场景、首条消息、创作者备注。
- 世界书标题、关键词、正文、启用状态。
- 删除不需要的角色草稿或世界书条目。

### 校验与警告

- JSON / Markdown fenced JSON 解析。
- 轻量结构校验。
- 可恢复字段规范化。
- 空结果检测。
- token 粗估和预算提醒。
- 重复角色名、重复世界书标题、重复关键词、重复正文检测。

### 导出

支持导出：

- 完整内部草稿 JSON。
- 角色草稿 JSON。
- 世界书草稿 JSON。
- SillyTavern 兼容角色 JSON。
- SillyTavern 兼容世界书 JSON。

### 安全创建

支持创建：

- 创建到酒馆世界书。
- 创建到酒馆角色。
- 创建角色后，显式可选绑定本次新建世界书。

安全策略：

- 不覆盖已有角色。
- 不覆盖已有世界书。
- 不删除已有数据。
- 写入前创建本地备份记录。
- 写入后校验旧角色或旧世界书摘要仍存在。
- 绑定失败使用独立错误码 `E013`，不把角色创建成功误判为失败。

### 原生能力交接

创建报告会提示后续操作交给 SillyTavern 原生界面：

- 世界书后续编辑、移动、复制、删除交给原生世界书管理器。
- 角色后续编辑、头像、分组、删除、管理交给原生角色面板。
- 如果 `reloadWorldInfoEditor(name, true)` 可用，插件会提供原生世界书入口。
- 如果原生入口不可用，插件会显示手动查找提示。

## 日志功能

插件有详细日志功能，分为运行时日志和诊断导出。

### 记录位置

- 控制台：通过 `console.info`、`console.warn`、`console.error` 输出，前缀为 `[setting-organizer]`。
- 浏览器 localStorage：键名为 `setting-organizer.runtimeLogs.v1`。
- 最多保留 200 条日志，新日志排在前面。

### 日志级别

- `debug`
- `info`
- `warn`
- `error`

### 主要日志事件

常见事件包括：

- 扩展加载。
- 设置读取和保存。
- 模型调用完成或失败。
- 聊天读取完成或失败。
- 导出完成或失败。
- 备份创建。
- 世界书创建完成或失败。
- 角色创建完成或失败。
- 角色世界书绑定完成或失败。
- 原生世界书入口打开。
- 诊断日志导出。

### 脱敏策略

日志会自动脱敏这些字段：

- API key
- authorization / auth
- Cookie
- CSRF
- token
- secret
- password
- headers

prompt、聊天正文、世界书正文、角色描述等长文本不会完整保存，只记录长度和短预览。

### 诊断导出

面板提供：

- `导出诊断日志`
- `清空诊断日志`

导出的诊断 JSON 包含：

- 生成时间。
- 当前 URL。
- userAgent。
- SillyTavern 版本。
- 额外上下文。
- 最近运行日志。

日志不会自动上传，只有用户点击导出时才会生成本地 JSON 文件。

## 错误提示

页面不再只显示错误码。插件会显示：

- 人类可读的错误标题。
- 当前错误说明。
- 可操作处理建议。
- 保留错误码，方便搜索文档和定位日志。
- 必要时显示技术详情或底层原因。

错误码仍然保留：

| 错误码 | 含义 |
| --- | --- |
| `E001` | 模型调用失败 |
| `E002` | 模型输出不是合法 JSON |
| `E003` | 结构校验失败 |
| `E004` | 分析结果为空 |
| `E005` | token 超过预算 |
| `E006` | 导出失败 |
| `E007` | 备份失败 |
| `E008` | 角色创建失败 |
| `E009` | 世界书创建失败 |
| `E010` | 当前 SillyTavern API 不兼容 |
| `E011` | 旧数据摘要校验失败 |
| `E012` | 当前聊天读取失败 |
| `E013` | 角色世界书绑定失败 |
| `E014` | 输入内容无效 |
| `E015` | 输入内容为空 |
| `E016` | 已取消超长输入分析 |

## 已知限制

- 真实模型输出质量取决于当前 SillyTavern 模型和 prompt 遵循情况。
- 世界书、角色和绑定接口已在 SillyTavern 1.18.0 验证，其他版本需要先做兼容性测试。
- 备份是手动恢复依据，不是完整自动回滚。
- 重复检测是规则级检测，不做复杂语义推理。
- 不支持完整 RAG、关系图、高级冲突推理或完整角色/世界书管理器。
