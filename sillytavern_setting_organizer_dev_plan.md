# SillyTavern 设定整理与一键导入扩展开发计划

> 说明：本文档为原始总体规划。实际交给 Agent 执行时，以 `setting_organizer_agent_dev_plan.md` 和 `setting_organizer_task_cards.md` 中的分阶段计划为准。
> 第一版不承诺完整自动回滚，只承诺创建备份、失败状态报告和可手动恢复依据；完整自动回滚需在后续版本实际实现并验证后再启用。

## 一、项目目标

开发一个适用于 SillyTavern 的第三方扩展，用于将零散设定、角色资料、剧情记录和聊天内容自动整理为：

- 角色卡
- 世界书
- 用户人设
- 作者注释
- 动态剧情状态
- 剧情摘要
- 可选的 Data Bank / RAG 资料

扩展应支持预览、编辑、冲突检测、Token 预算、版本备份和一键导入，避免直接覆盖已有内容造成数据损坏。

---

## 二、核心使用流程

用户操作流程：

1. 打开扩展面板。
2. 选择资料来源。
3. 选择整理目标。
4. 选择整理模板和 Token 预算。
5. 调用大模型分析。
6. 查看分类结果、冲突和重复内容。
7. 手动修改或锁定部分内容。
8. 预览即将写入的变化。
9. 确认导入。
10. 自动保存备份和版本记录。

标准数据流程：

```text
原始资料
→ 文本预处理
→ AI结构化分析
→ JSON格式校验
→ 规则校验
→ 冲突与重复检测
→ 用户预览编辑
→ 创建备份
→ 写入SillyTavern
→ 保存版本记录
```

## 三、开发范围

### 第一阶段：最小可用版本 MVP

第一阶段目标是实现一个安全、稳定、能实际使用的基础版本。

为降低实现风险，第一阶段拆分为：

- MVP-A：粘贴文本、AI 整理、JSON 校验、人工编辑、JSON 导出。
- MVP-B：备份、导入确认、创建新世界书、创建新角色、旧数据不变验证。
- MVP-C：当前聊天读取、Android / Termux 兼容性强化。

主要功能：

- 扩展界面
- 粘贴文本输入
- 调用当前酒馆连接的大模型
- 自动生成角色卡字段
- 自动生成世界书条目
- 自动生成世界书关键词
- JSON 结果校验
- 结果预览和编辑
- 导出角色卡 JSON
- 导出世界书 JSON
- 导入前自动备份
- 创建新角色或新世界书
- 默认不覆盖现有数据

MVP-C 再实现：

- 读取当前聊天内容

第一阶段暂不实现：

- 自动持续更新
- 多项目同步
- RAG 文件管理
- 高级冲突推理
- 多角色关系图
- 完整版本差异对比
- PNG 角色卡封装

### 第二阶段：设定管理增强

增加长期剧情和复杂项目管理能力。

主要功能：

- 项目制管理
- 永久设定与动态状态分离
- 角色、NPC、势力、地点分类
- 重复条目检测
- 设定冲突检测
- Token 预算管理
- 世界书关键词风险检测
- 局部重新生成
- 内容锁定
- 合并和拆分世界书条目
- 修改前后差异预览
- 自动版本快照
- 一键恢复旧版本

### 第三阶段：长篇剧情管理

主要功能：

- 从聊天记录提取剧情变化
- 自动提取未完成任务
- 自动提取人物关系变化
- 自动提取物品变化
- 自动提取秘密及知情范围
- 动态剧情状态更新
- 历史剧情分段摘要
- 旧剧情压缩
- 人物状态追踪
- 时间线检查
- 多角色卡批量生成

### 第四阶段：高级功能

主要功能：

- Data Bank / RAG 文件整理
- 项目完整打包与恢复
- PNG 角色卡导出
- 世界书关系图
- NPC 关系网络
- 设定来源追踪
- 多模型整理模板
- 多语言支持
- 自动测试与诊断
- 项目导入迁移工具

## 四、技术方案

### 1. 开发形式

采用 SillyTavern 第三方前端扩展形式。

推荐技术：

- JavaScript
- HTML
- CSS
- SillyTavern 扩展接口
- 浏览器本地存储
- JSON Schema 校验
- 可选使用轻量前端组件库

不建议第一版使用：

- Python 后端
- 独立数据库服务
- Docker

- 单独启动的本地服务器

## 六、AI 整理方案

### 1. 第一轮：内容识别

识别内容中的：

- 角色
- NPC
- 地点
- 势力
- 物品
- 能力
- 世界规则
- 时间线
- 人物关系
- 剧情事件
- 当前状态
- 禁止事项
- 文风要求

### 2. 第二轮：分类整理

将识别内容分配到：

- 角色卡
- 世界书
- 用户人设
- 作者注释
- 动态状态
- 历史摘要
- 长篇参考资料

### 3. 第三轮：校验

检查：

- 是否缺少必填字段
- 是否出现重复条目
- 是否出现设定冲突
- 关键词是否过于宽泛
- 常驻内容是否过多
- 是否超过 Token 预算
- 是否把动态状态误写入永久设定
- 是否把用户设定误写入角色卡

### 4. 模型输出要求

模型必须输出严格 JSON，不直接输出自然语言解释。

推荐返回结构：

```json
{
  "characters": [],
  "lorebookEntries": [],
  "persona": null,
  "authorNote": "",
  "dynamicState": {},
  "conflicts": [],
  "warnings": [],
  "tokenEstimate": {}
}
```

程序必须对模型输出进行：

- JSON 解析
- Schema 校验
- 字段缺失补全
- 类型修正
- 非法内容拦截
- 错误重试

## 八、安全设计

必须遵守以下原则：

1. 默认不覆盖。
2. 默认不删除。
3. 默认不修改全局世界书。
4. 导入前必须创建备份。
5. AI 不能直接执行写入操作。
6. 写入前必须经过程序校验。
7. 所有修改必须可撤销。
8. 锁定内容不得被 AI 修改。
9. 模型输出异常时禁止导入。
10. 大批量操作必须二次确认。

## 九、Token 预算设计

预设四种模式。

### 轻量模式

- 角色卡：1200 Tokens
- 常驻世界书：700 Tokens
- 单条世界书：200 Tokens
- 作者注释：150 Tokens
- 动态状态：400 Tokens

### 标准模式

- 角色卡：2000 Tokens
- 常驻世界书：1200 Tokens
- 单条世界书：350 Tokens
- 作者注释：250 Tokens
- 动态状态：800 Tokens

### 长篇模式

- 角色卡：3500 Tokens
- 常驻世界书：2500 Tokens
- 单条世界书：600 Tokens
- 作者注释：400 Tokens
- 动态状态：1500 Tokens

### 自定义模式

用户自行填写各模块限制。

## 十、开发里程碑

### 里程碑一：扩展骨架

完成：

- 扩展加载
- 设置面板
- 基础 UI
- 本地配置保存
- 调试日志

验收标准：

- 扩展可正常安装
- 在 Android Termux 环境下可打开
- 页面刷新后设置仍然保留

### 里程碑二：AI 结构化整理

完成：

- 文本输入
- 调用当前模型
- JSON 结构化输出
- Schema 校验
- 错误提示

验收标准：

- 输入一段设定后能生成合法角色和世界书数据
- 模型返回非 JSON 时不会导致扩展崩溃
- 可重新发起整理

### 里程碑三：结果编辑器

完成：

- 角色卡编辑
- 世界书条目编辑
- 关键词修改
- 条目启用和禁用
- 局部删除
- JSON 导出

验收标准：

- 所有结果均可人工修改
- 修改后导出数据正确
- 页面切换不会丢失内容

### 里程碑四：安全导入

完成：

- 创建备份
- 创建角色
- 创建世界书
- 绑定世界书
- 导入确认
- 失败状态报告
- 可手动恢复依据

验收标准：

- 不修改原角色也能创建副本
- 导入失败时能显示已完成步骤、未完成步骤、可能影响范围和备份标识
- 导入前后能证明旧角色和旧世界书未变化
- 备份可作为手动恢复依据

### 里程碑五：冲突与 Token 检查

完成：

- 重复检测
- 基础冲突检测
- Token 估算
- 关键词风险提示
- 常驻内容占用统计

验收标准：

- 能识别明显重复条目
- 能提示明显矛盾
- 能显示预计上下文占用

### 里程碑六：项目和版本管理

完成：

- 创建项目
- 保存项目
- 项目切换
- 自动快照
- 版本恢复
- 项目导出和导入

验收标准：

- 不同项目数据互不干扰
- 可以恢复到之前版本
- 可完整迁移到另一台设备

## 十二、错误处理

建议统一错误分类：

```text
E001 模型调用失败
E002 模型输出不是合法JSON
E003 JSON Schema校验失败
E004 结果为空
E005 Token超过预算
E006 导出失败
E007 备份创建失败
E008 角色创建失败
E009 世界书创建失败
E010 当前酒馆版本不兼容
E011 旧数据完整性校验失败
E012 当前聊天读取失败
```

每个错误应提供：

- 错误原因
- 影响范围
- 是否已回滚
- 可执行的处理方式
- 调试日志复制按钮

日志分为：

- 普通信息

- 操作类型

- 输入长度
- 输出长度
- Token 估算

## 十八、结论

项目应先完成一个安全的最小可用版本：

```text
粘贴设定
→ AI自动分类
→ 生成角色卡和世界书
→ 人工预览
→ 自动备份
→ 一键导入
```

第一版的核心不是功能最多，而是：

- 不破坏原数据
- 整理结果可人工修改
- 模型输出可校验
- 所有操作可撤销
- 能在 Android Termux 环境稳定运行

在基础版本稳定后，再增加项目管理、动态剧情、冲突检测、版本恢复和 RAG 功能。
